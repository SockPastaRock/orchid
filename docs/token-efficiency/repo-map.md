# Repo Map

**Category:** Compression  
**Expected impact:** Medium  
**Risk:** Low

## Problem

At the start of a coding task, agents read whole files to understand the codebase structure — what exists, where things are, what a function signature looks like. Most of that file content is irrelevant to the actual task but still enters context.

## Approach

Inject a compact, repo-wide structural summary at conversation start. The map contains directory structure plus symbol exports (function names, type names, signatures) without bodies:

```
cmd/orchid/
  main.go         run(args []string) error
  send.go         cmdSend(args []string) error
internal/loop/
  loop.go         New(...) *Runner
                  (r *Runner) Run(ctx, meta, systemPrompt, userMessage, opts) error
internal/provider/
  provider.go     type Provider interface { Send(...) }
                  type Request struct
                  type Response struct
```

This gives the model enough to navigate and make targeted reads rather than exploratory ones.

## Storage

Cached as a `repo_map` field in `metadata.json`. Regenerated by an `on_run_start` hook when the working directory has changed since last generation.

## Notes

- Generation can use static analysis or tree-sitter — no model call required
- Pairs well with [local background model](local-background-model.md) for lightweight regeneration
- The map should be bounded — very large repos may need filtering to the most relevant packages
